## Challenges & Growth Areas

1. **Stale-state reads pattern** — appeared in 3/6 stories (3.2, 3.3, 3.4) before becoming automatic. Developer should capture state fields into locals before `await` without needing code review to flag it
2. **BlocConsumer no-op listeners** — pre-existing antipattern found in 4/6 stories. Systematic fix applied but detection was reactive, not proactive
3. **Delay placement bugs** — incorrect `Future.delayed` placement relative to emit calls in 2/6 stories (3.3, 3.4)
4. **Code review found test coverage gaps in 4/6 stories** — developer tends to cover happy path but misses edge cases (null stackTrace, stream errors, validation boundaries)
5. **==`/`hashCode required on every data-carrying state variant** — needed in every single story but not yet automatic for the developer
6. **project-context.md not updated during brownfield modernization** — agents received outdated information about SDK versions, folder structure, and patterns

## Key Insights & Learnings

1. **Deferred item system works when there's a dedicated cleanup story** — Story 3.6 was planned from the start as the "catch-all" for accumulated items. Every epic should have this.
2. **project-context.md must be updated at each retro closure in brownfield** — the document was created based on pre-modernization code and became increasingly misleading through 3 epics
3. **Dartdoc comments are part of professional quality** — identified as a gap during retro. Codebase lacks professional documentation comments on classes, widgets, use cases. `public_member_api_docs` was disabled in analysis_options.yaml.
4. **"Accepted" should be last resort, not default** — Christopher's directive: if it can be improved, improve it. Only accept when there's irrefutable technical justification.
5. **The "right moment" for a fix matters** — fixing a no-op listener in Story 3.1 when that widget gets rewritten in Story 3.2 is wasted effort. But the fix must not be forgotten.
6. **Spec versions are hypotheses — validated again** — bloc 8.x vs 9.x deduplication behavior discovery in Story 3.5 confirmed this principle from Epic 2
